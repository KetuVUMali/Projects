What is data-aos-mirror?

Mirror decides what happens when user scrolls UP.

✔ mirror = true → animation runs again while scrolling UP ✔ mirror = false → animation runs only while scrolling DOWN

This is very important for UX.

Scroll down first, then scroll up slowly

Scenario 1: mirror = false (Recommended)

Animation triggers when scrolling DOWN.
When scrolling UP → NO animation.

✔ Best for content, text, images.

Fade-Up (No animation on scroll UP)

Scenario 2: mirror = true

Animation works BOTH ways.
Scroll DOWN → animation Scroll UP → animation again

✔ Useful only in special cases.

Animation on DOWN & UP

Bad Practice Example

Text repeating animation while scrolling UP is confusing for users.
❌ Avoid mirror=true for reading content.

❌ Text animates again on scroll UP

Image / Gallery Section

In visual sections, mirror animation can feel nice.
✔ Allowed for creative UI.

Gallery Image (Mirror enabled)

Dashboard Widgets

Widgets may animate again to show interaction.
✔ mirror=true is acceptable.

Dashboard Card (Two-way animation)

Hero Section (Strict Rule)

Hero animation should NEVER repeat.
✔ mirror=false is mandatory.

Hero Heading (No reverse animation)
