What is data-aos-once?

data-aos-once controls:

👉 Should animation run ONLY ONE TIME?
👉 OR should it run EVERY TIME user scrolls?

true → animation runs only once false → animation repeats on scroll

Scroll DOWN, then scroll UP to see the difference clearly

Default Behavior (data-aos-once = false)

By default, AOS animations repeat.
When you scroll up and down, animation runs AGAIN.

❌ Can feel distracting if used everywhere.

Animation repeats on every scroll

Content Section (data-aos-once = true)

Text content should animate ONLY ONCE.
Repeating text animation feels unprofessional.

✔ This is STANDARD company practice.

Content animates only one time

Hero Section (Welcome Area)

Hero animation should happen ONLY ON FIRST LOAD.
User scrolls back → animation should NOT repeat.

Hero Heading (One-time animation)

Services Cards – Once = True

Cards should animate once to introduce content.
Repeating card animation looks noisy.

Dashboard Widgets (once = false)

In dashboards, repeated animation can give feedback on scroll.
✔ Acceptable use case.

Image / Media Section

Images should animate ONCE to avoid distraction.
Especially important for portfolios.
