Basic Delay Concept

data-aos-delay means:
"Wait for some time AFTER element becomes visible, then start animation."

Time unit = milliseconds (ms)

No Delay (Starts immediately)
300ms Delay (Starts after 0.3 sec)
600ms Delay (Starts after 0.6 sec)

Hero Section – Text Flow Animation

In real websites, Hero section text should appear in reading order:
Heading → Subheading → Button

Hero Heading (0ms delay)
Hero Subheading (200ms delay)
Call To Action Button (400ms delay)

Services Cards – Stagger Effect

This is the MOST COMMON real-world usage of data-aos-delay.
Cards should NOT animate together. They should appear one by one.

Service Card 1 (0ms)
Service Card 2 (200ms)
Service Card 3 (400ms)

Image & Text Sync Animation

Common layout: Image first → then explanation text. Delay helps user focus step by step.

Image / Illustration (0ms)
Description Text (300ms)

Testimonials – Gentle Entry

Testimonials must feel calm & trustworthy. Slight delay avoids sudden appearance.

Very high delay makes UI feel broken. This example is ONLY to understand limits.
