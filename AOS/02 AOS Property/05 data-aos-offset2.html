Basic Concept of data-aos-offset

data-aos-offset controls
HOW EARLY or HOW LATE the animation starts when you scroll.

Offset value = distance (in pixels) from bottom of screen
BEFORE animation triggers.

Default Offset (120px)
Small Offset (50px → animation starts late)
Large Offset (300px → animation starts early)

Hero Section – Early Trigger

In real websites, hero content should animate BEFORE it fully appears.
That creates premium feel.
Companies use HIGH offset here.

Hero Heading (offset 400px)
Hero Subheading (offset 350px)

About Section – Normal Reading Flow

Informational content should animate when user actually reaches it.
Default offset is BEST here.

About Text (offset 120px – default)

Services Cards – Balanced Offset

Cards should animate slightly early, but not too early.
Medium offset gives clean UX.

Service 1 (offset 200px)
Service 2 (offset 200px)
Service 3 (offset 200px)

Image & Text – Focus Control

Image should animate first, text should animate when image is visible.
Different offset helps focus user attention.

Image / Illustration (offset 300px)
Supporting Text (offset 150px)

Statistics Section – Late Trigger

Numbers should animate only when clearly visible.
Smaller offset is better.

Very High Offset – Avoid

Too large offset can trigger animation when element is NOT visible.
This feels confusing to users.

Offset 800px (❌ Not Recommended)
