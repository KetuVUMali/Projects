What is data-aos-easing?

Easing controls HOW animation moves over time.

👉 Not WHEN it starts 👉 Not HOW LONG it runs 👉 Only HOW it accelerates & decelerates

Think like driving a bike:
Sudden speed ❌ | Smooth pickup ✅ | Smooth stop ✅

Scroll slowly to FEEL the difference between easing types

linear (Robotic Movement)

linear means:
Same speed from start → end.

❌ Looks robotic ❌ Not natural ❌ Rarely used in real projects

Linear Easing (Constant Speed)

ease (Default & Balanced)

ease is browser default.
Slight slow start → fast middle → slow end.

✔ Safe choice ✔ Balanced ✔ Used when no custom UX needed

ease (Default behavior)

ease-in (Slow Start)

ease-in means:
Starts slowly → ends fast.

✔ Good for elements entering quietly ✔ Used for secondary content ❌ Bad for CTA

ease-in (Slow Start)

ease-out (Smooth Ending)

ease-out means:
Starts fast → ends slowly.

✔ Best for cards ✔ Best for content reading ✔ Feels natural

ease-out (Smooth Stop)

ease-in-out ⭐ (Company Standard)

ease-in-out means:
Slow start → smooth middle → slow end.

⭐ MOST RECOMMENDED ⭐ Used in almost all professional websites ⭐ Safe for every section

ease-in-out (Perfect Balance)
ease-in-out (Images / Cards)

Custom Cubic Bezier (Advanced)

Custom easing gives full control.
Used only when designer demands special motion.

❌ Not for beginners ✔ Optional advanced usage
