GSAP Timeline with Overlap Example : Overlap using "-=time"

This example demonstrates a GSAP timeline where a box moves and rotates with an overlap in the animations.

  • The box first moves 500px to the right over 4 seconds.
  • Then, it starts rotating 180 degrees over 1 second, beginning 0.5 seconds before the move animation ends.

This showcases how timelines can be used to create overlapping animations effectively.

"-=time" means start this animation 'time' seconds before the previous one ends. "+=time" means start this animation 'time' seconds after the previous one ends. "0" means start this animation exactly when the previous one ends. "" (a number) means start this animation at an absolute time in the timeline. "=time" 
    🧠 Explanation (Step-by-step)
                    Timeline created
                    Box moves right for 4 seconds
                    After 3.5 seconds (0.5s before move ends) → box starts rotating
                    Overlap creates smooth transition
        
             tl.to(".box", { x: 500, duration: 4 })                 // Box moves right for 4 seconds
                                                                    // After 3.5 seconds, start next animation
            .to(".box", { rotate: 180, duration: 1 }, "-=0.5");     // Box rotates 180 degrees over 1 second, starting 0.5 seconds before move ends
        
            Explanation Summary:
                    Box starts moving right
                    Rotation starts 0.5 seconds before move ends
                    Motion feels natural & professional
