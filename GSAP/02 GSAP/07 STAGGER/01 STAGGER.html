STAGGER – LIST ANIMATION 📌 Concept Animate multiple elements with delay between each. GSAP Stagger 
      STAGGER:
      Simple meaning:

      Stagger = delay animation of multiple elements one by one
                Delay all elements individually
                time gap between start of one element's animation and the next element's animation.

      *duration* : take time to start animation. before start take time and perform all animation at a time.
                    Take time to perform animation. (Complete animation duration)
                  Then stagger delays start of each element's animation by specified time.
      

      Instead of:
      All elements animate together ❌
      Animate one element
      then next element
      then next element
      then next element

      We get:
      Item 1 animates
      then Item 2
      then Item 3
      then Item 4 ✅

GSAP Stagger Example : List Animation

This example demonstrates the use of GSAP's stagger property to animate multiple elements in a sequence.

  • Four blue cards are animated to move up by 40 pixels and fade in from opacity 0.
  • The stagger property is set to 0.2 seconds, causing each card's animation to start 0.2 seconds after the previous one.
  • The duration of each animation is set to 0.6 seconds.

This showcases how the stagger property can be used to create appealing sequential animations for lists or groups of elements.

    🧠 Explanation (Step-by-step)
                    Select all .card elements
                    Animate from y:40 and opacity:0
                    Stagger each animation by 0.2s
                    Each animation lasts 0.6s
                    Creates a sequential entrance effect
